The **Runs** tab is where you start new detection work and where you go to see what Conway has been doing. It's the default tab when you open Detection.

## Creating a run

Runs are always scoped to a specific table. Click **DRAFT RUN** in the top right and pick a source → database → schema → table. Click **LAUNCH RUN** to kick it off — Conway moves the run into the **In Flight** group and starts executing.

<Steps>
  <Step title="Click DRAFT RUN">
    In the top right of the Runs tab.
  </Step>
  <Step title="Pick your data">
    Only the sources and tables your deployment has access to appear in the cascade.
  </Step>
  <Step title="Click LAUNCH RUN">
    Conway moves the run into **In Flight** and starts executing.
  </Step>
</Steps>

## The run feed

Below the create button, the feed lists every past run grouped by age — in-flight at the top, the last seven days next, everything older collapsed by default (click **Show {N} more** to expand). In-flight cards animate and show the phase the agent is currently in, so you can glance at the feed and tell what's still running.

Search the feed by objective, ID, or source; narrow by status (processing / completed / failed) or by verdict (Positive / Negative / Uncertain). Click any run to open its full detail view.
