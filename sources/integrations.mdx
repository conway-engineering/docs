The **Integrations** tab on the Sources page is the admin surface for wiring Conway up to third-party services — Slack for RFIs, identity providers, ticketing, and similar.

<Note>
This tab is **visible to admins only**. Regular analysts see *"Contact your administrator to manage integrations."* when they click it. This page is for admin reference; day-to-day analyst work doesn't require it.
</Note>

## The integrations list

Each integration is one row. You'll see its service name, connection state (connected, disconnected, or error), and when it was last used. Use the row's controls to edit credentials, test the connection, or remove it.

## Adding an integration

Click **Add integration** and pick the service. Most use OAuth; a few need an API token supplied inline. The integration becomes available immediately to the surfaces that use it — for example, connecting Slack makes **Slack** appear as a channel option in the RFI dialog.

## Common integrations

- **Slack** — used as an RFI channel. Without it, analysts can't pick Slack when sending a request for information.
- **Identity provider (via WorkOS)** — controls who can log in. Managed through the **WORKOS PORTAL** button in the Settings modal.
- **Ticketing** — wires the **Ticket** channel in the RFI dialog to a ticketing system.

## Handing off admin duties

If you're transferring admin responsibility to someone else, walk them through this tab so they know which integrations are currently live and which would need reconfiguration. Only the admin or owner role should be making changes here.
