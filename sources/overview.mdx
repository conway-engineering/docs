**Sources** is where Conway's connection to your data lives. Every run is scoped to a source, and every source comes from this page.

The page has two tabs — **Warehouses** (your data warehouses, which detection reads from) and **Integrations** (admin-only, for third-party services). Analysts spend their time in Warehouses; Integrations is documented separately in [Integrations](/sources/integrations).

## Warehouse cards

Each connected warehouse appears as a card. A health indicator tells you whether the connection is currently working; the card also records the warehouse type (Snowflake / Redshift) and the authentication method in use (OAuth / key pair / password).

From the card, you can:

- **Test source** — confirm Conway can still authenticate and list tables. Always run this after anything changes — see [Testing a connection](/sources/testing).
- **Open query console** — jump into the [query console](/sources/query-console) preselected on this warehouse.
- **Edit scope** — change the default database/schema Conway operates on.
- **Remove** — disconnect the warehouse.

## Connecting a new warehouse

Click **Add Data Source** and pick a type. Each warehouse type has its own connection flow:

- [Connecting Snowflake](/sources/snowflake)
- [Connecting Redshift](/sources/redshift)

After connecting, run a [test](/sources/testing) before kicking off any live runs.

<Note>
In **Pilot deployments**, your sources are preconfigured for you. See [Sources configured for you](/pilot/sources-configured). Changes to scope or new connections are coordinated with the Conway team rather than self-service.
</Note>

## Integrations visibility

The Integrations tab is admin-only. Analysts without admin privileges see *"Contact your administrator to manage integrations."* — if that's you, this is expected, and you don't need the tab for day-to-day work.
