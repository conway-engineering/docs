The **query console** lets you run SQL against any connected warehouse directly inside Conway. Whatever the agent can see, you can see.

## Using the console

Open the console from a source card on the [Sources](/sources/overview) page — click **Open query console** and the console opens with that source preselected. If you have more than one source connected, switch between them from the selector at the top.

Browse tables on the left rail before you write anything. Click a table to preview its columns; click **Insert SELECT** and Conway pastes a starter query for it into the editor.

Write your SQL in the editor and click **Run** (⌘/Ctrl+Enter). Results appear below with a row count, sortable and resizable.

## Reproducing a query from a run

When you're reading a run's [Telemetry](/run-detail/telemetry) tab and want to see a cited value with your own eyes, copy the SQL from the telemetry entry and paste it in here. This is the fastest way to sanity-check something the agent claimed.

## Testing a rule proposal

When a candidate rule shows up under **PROPOSALS** in [Rules](/detection/rules), paste its SQL into the console to see what it returns against today's data before you accept or reject it.

## Tips

- Conway runs queries with the role and scope of the source's saved credentials. If a table is not visible in the console, the agent cannot see it either.
- Results are capped so the browser stays responsive. If you need more rows, add a `LIMIT` and narrow the query.
