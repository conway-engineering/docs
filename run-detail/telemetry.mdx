The **Telemetry** tab is the audit trail for a run — every tool the agent invoked, every query it ran, and every intermediate result, in the order it happened. Use it when you want to understand *why* the agent reached a verdict step by step. For a quick "what did the agent find?", the [Investigations tab](/run-detail/investigations) is faster.

## The telemetry feed

While a run is processing, entries stream in from the bottom and the feed auto-scrolls. Scroll up to pause the auto-scroll and read past steps without losing the newest ones.

Click any entry to expand it. A **Query** step shows the SQL and a preview of the result. An **Analysis** step shows the agent reasoning over data it just pulled. An **Investigation** step marks the moment the agent opened a new investigation — the matching report appears on the [Investigations tab](/run-detail/investigations). An **Intelligence** step records something the agent chose to remember, visible on the [Intelligence tab](/run-detail/memories). Phase transitions are marked in line, so you can see where one phase of the run ended and the next began.

## Reproducing a query

When a step shows SQL, copy it, open the [query console](/sources/query-console), and paste it in. This is the shortest path from "the agent says X" to "I confirmed X against the warehouse myself."

## Debugging a failed run

If a run ended in **Failed**, scroll to the last entry. The step where things went wrong is almost always the bottom of the feed — expand it for the error and the inputs that produced it.
